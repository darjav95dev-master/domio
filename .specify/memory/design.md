## 0. Identity

- **Project:** Domio
- **Page-job this design serves:** Three public surfaces — Home (`/`), Portafolio (`/portafolio`), Detalle de inmueble (`/inmuebles/<slug>`) — plus contact and private backoffice. Common job: that the visitor perceives Domio as a **serious, editorial real-estate house** — a curated portafolio of commercial and residential inmuebles in the Canary Islands, transparent with prices and specs, never a corporate template nor a hype-driven marketplace.
- **Voice (3 adjectives):** Editorial, Confiable, Cálido

- **Generated on:** 2026-07-05

## 1. The signature

**Numerales Fraunces itálicos como anclas estructurales.** The gesture that makes this design recognizable: every relevant number — steps, metrics, payment plazos, progress counters, precios, superficie en m², año de construcción — carries a large numeral set in **Fraunces italic**. Not as decoration, but as the **"editorial seal"** of a house that treats numbers with dignity — a real-estate portafolio that publishes its data proudly, not hidden in fine print.

The parchment stack (warm cream on dark ink) and the single terracota accent amplify that character: **ledger-editorial**, never startup-fintech, never corporate-blue.

How it appears across the system:

- **On the hero:** `trust-card` with `font-family: serif; font-size: 26px italic` for the three trust datapoints (años en el mercado, inmuebles gestionados, ciudades).
- **On how-it-works / process steps:** `.how-num` — italic numerals `52px` as the title of each step.
- **On metrics / trust bands:** `.metric-num` — `72px`, `letter-spacing: -.04em`, with the unit in Geist Mono as a superscript.
- **On payment plans / financing steps:** `32px italic serif` for each importe.
- **On detail / progress:** `80px terracota italic` for percentage occupied / percentage sold / disponibilidad.
- **Never appears in:** buttons, navigation, input labels, footer legalese. The italic numeral is a **content-level** gesture, not chrome.

Borrowed from long-form editorial typography and old real-estate ledgers: **authority through composure and typography**, not through restraint alone, not through hype. Where a corporate real-estate site whispers in blue-and-gray, Domio speaks in warm ink with a single terracota mark.

- **Masthead (Home):** full-bleed architectural photograph with warm gradient overlay, hero copy bottom-left, trust-card bottom-right with three italic numerals.
- **Portafolio (`/portafolio`):** paper band, filter bar, dense modular grid of property cards.
- **Detalle (`/inmuebles/<slug>`):** photo-hero + infobar with 4 tabular datapoints (precio/m², superficie, plazas, entrega) each as a `numeral-lg` italic, then split body `1.6fr 1fr` with sticky aside.

### 2.3 Contrast commitments

> Verified against WCAG 2.1 relative-luminance. If a pair fails, the palette is revised — not the commitment.

| Pair | Ratio | Required | Result |
|------|-------|----------|--------|
| `fg.default` #1A1410 on `bg.canvas` #FBF8F3 | 17.2:1 | ≥7:1 (AAA) | PASS |
| `fg.default` #1A1410 on `bg.surface` #FFFCF7 | 17.8:1 | ≥7:1 | PASS |
| `fg.muted` #3D2E20 on `bg.canvas` #FBF8F3 | 12.3:1 | ≥4.5:1 (AA) | PASS |
| `fg.subtle` #7A6857 on `bg.canvas` #FBF8F3 | 5.0:1  | ≥4.5:1 (AA floor) | PASS |
| `fg.on-inverted` #FFFCF7 on `bg.inverted` #2E2B27 | 13.8:1 | ≥7:1 | PASS |
| `on-dark-em` #FFD9B0 on `bg.inverted` #2E2B27 | 10.6:1 | ≥4.5:1 | PASS |
| `focus-ring` #C75D3F on `bg.canvas` | 3.9:1  | ≥3:1 | PASS |

**Critical rules:**
- `warm-amber` #FFD9B0 is used **ONLY as em italic on dark backgrounds** (hero H1, CTA H2, footer tagline). Never on light — it fails AA there.
- `fg.brand` #C75D3F (terracota) does **not** appear in the table above because it must **never** carry body text on `bg.canvas` — actual ratio is 3.9:1, which fails AA (≥4.5:1). Terracota expresses the accent through **fills, borders, icons, progress bars, and focus rings**, not through text on light surfaces. For large text (≥18px) on dark surfaces (`bg.inverted`, `bg.band-ink`) where ≥3:1 suffices, `fg.brand` may be used sparingly — verify the specific pair before committing.

## 3. Typography

### 3.1 Families

```yaml
font.family:
  display: "'Fraunces', 'Times New Roman', Georgia, serif"
  body:    "'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif"
  mono:    "'Geist Mono', ui-monospace, SFMono-Regular, Menlo, monospace"
```

- **Fraunces** (display + italic): serif with variable optical-size (`opsz 9..144`) and true italics. Carries **all** display and heading roles, plus the italic numeral signature. Weights 400 / 500 / 600 / 700. Italics only 400i / 500i. The optical-size axis makes it sit properly at both 26px (trust-card) and 96px (hero H1).
- **Inter** (body/UI): neutral workhorse for body text, labels, controls, form inputs. Weights 300 / 400 / 500 / 600 / 700.
- **Geist Mono**: metadata — precios, referencia catastral, año, m², contadores, pagination, table headers. `font-variant-numeric: tabular-nums`.

**Pairing logic:** editorial serif display + neutral sans body is the **ledger-magazine** spine; italic Fraunces provides the tonal accent that keeps the system from reading cold or corporate. Emphasis inside a serif heading is the **serif italic itself** (Fraunces italic), same visual size — never a switch to sans or bold.

**Notas de carga:** todas vía `next/font/google` con `display: 'swap'`. Ver §12.5.

Ratio: ~1.33 (perfect fourth — magazine cadence). Base body: 16px.

### 3.5 Rules

- Components reference **roles**, never raw sizes.
- `font-variant-numeric: tabular-nums` globally on precios, m², años, porcentajes, contadores, ISBNs, pagination.
- Fraunces carries **all** display and heading roles. Inter is body/UI only. There is no third font family.
- Emphasis (`<em>`) inside a Fraunces heading = Fraunces **italic** of the same visual size (weight 400). On dark backgrounds, `em` receives `color: warm-amber` (#FFD9B0). On light backgrounds, `em` inherits `fg.default`.
- `text-wrap: balance` on all `h1`, `h2`. `text-wrap: pretty` on body ≥ 2 lines.
- `display-xl` used ≤ 1 per page. `display-lg` ≤ 1 per page (CTA final). `display-md` ≤ 3 per page.
- Body blocks capped at `62ch`. Lead capped at `52ch`.
- **The eyebrow** carries the signature `::before` line: 32px width, 1px height, `linear-gradient(90deg, terracota → transparent)`. That mark is the eyebrow's tell.

## 6. Motion

```yaml
motion.duration:
  instant:    "0ms"
  quick:      "150ms"    # hover color changes
  standard:   "250ms"    # background transitions, nav items
  deliberate: "350ms"    # buttons, cards (transform + shadow)
  slow:       "500ms"    # card lift high-impact
  reveal:     "1200ms"   # scroll-reveal
  image:      "1400ms"   # image zoom on hover
  image-slow: "1600ms"   # coop image on card hover

motion.easing:
  standard: "cubic-bezier(.2, .8, .2, 1)"    # --ease: all UI transitions
  linear:   "linear"                          # marquee scroll
```

**Motion posture:** contained and warm — **nothing bounces, everything glides**. Image transitions are deliberately slow (1.4–1.6s) to give a sense of weight. Scroll-reveal uses staggered delay of `80ms` per child inside `.reveal-stagger`.

**`prefers-reduced-motion: reduce`:** `reveal` and `image` durations collapse to `instant`; `deliberate` and `slow` collapse to `quick`. Stagger delays removed. Marquee stops. Color transitions preserved.

## 8. Patterns

### Hero / Masthead (Home)
Full-bleed photograph (100vh min-height, `object-fit: cover`) with **multi-layer overlay**: radial gradient + linear gradient warm-ink + SVG fractal-noise texture at 0.5 opacity (grain). Inner grid `1.4fr 1fr` — copy bottom-left (H1 display-xl + eyebrow glass-pill + lead + 2 buttons), TrustCard bottom-right (3 italic numerals). Trust-marquee band at the very bottom (14px height, blurred glass, terracota icons).

### Section opener
`eyebrow` (mono uppercase with the 32px terracota-gradient `::before`) + `h2` in display-md + `.lead` text below. `.sec-head` capped at 760px. Centered variant for trust bands.

### How-it-works grid
4 columns with `gap: 1px`, `background: line-soft` (the gap is the separator, no explicit borders). Each cell: italic numeral (52px) as first visual, then icon-circle, then heading, then body.

2-column split `1fr 1.05fr`. Left: architectural photograph with a bottom-left overlay tag. Right: compare table where the "Domio" column carries a `border-top: 2px accent`. Row headers in Geist Mono uppercase.

### Testimonial band
`.sec-bone` background. `::before` quote mark `"` (U+201C) in Fraunces italic 140px terracota opacity 0.10 — typographic quota mark, not decoration. Card lifts on hover.

### CTA full-bleed
Photograph background + warm overlay + inner container aligned left (padding-left 96px, max-width 740px). H2 display-lg with `em` in warm-amber. Primary button (white bg → terracota on hover).

Border-top/bottom per item. `.plus` circular, rotates 45° on open. `max-height` transition for the panel. First item may open by default on `/faq`, never on `/`.

### Empty state
Composed — never bare text. Includes: eyebrow (context) + heading-sm (Fraunces) + body-sm (fg.subtle) + secondary button (recovery action). Example: "Todavía no hay inmuebles en esta categoría." + "Ver todo el portafolio".

### Loading state
Skeleton matches the real content dimensions (zero layout shift). Shimmer: `linear-gradient(90deg, paper-2 25%, paper 50%, paper-2 75%)` animated `bg-position`. `role="status"` + `aria-hidden="true"`. Reduced-motion: static `bg: paper-2`.

### Error state
Voice per §9 (affirmative, no blame). Visual: `border-left: 3px solid accent.default` (same treatment as the next-milestone banner). No stack traces exposed. No red background.

## 9. Voice in UI copy

Voice: **Editorial, Confiable, Cálido**.

- **Primary buttons:** verb + clear object. "Ver detalles", "Solicitar visita", "Descargar ficha técnica", "Contactar", "Acceder al área privada". Never "Click aquí", never "Enviar".
- **Secondary buttons:** more descriptive. "Ver todo el portafolio", "Conocer el proceso", "Ver documentos".
- **Section titles:** confident, short, editorial. "Portafolio destacado", "Cómo trabajamos", "Quiénes somos".
- **Eyebrow kickers:** mono uppercase with the terracota gradient line. Categorize genuinely — "Portafolio", "Proceso", "Confianza". Never decorative.
- **Errors:** affirmative, no blame. "No hemos podido enviar tu solicitud. Inténtalo de nuevo o llámanos al 922 000 000."
- **Empty states:** contextual + constructive. One sentence each, period at the end. "Todavía no hay inmuebles en esta zona." + action.
- **Confirmations:** brief and warm. "Solicitud recibida. Nuestro equipo te contactará en 24-48h."
- **Numbers with units:** always adjacent and tabulated (Geist Mono). "120 m²", "€1.850/mes", "3 plazas".
- **Case:** Sentence case throughout. Title Case only for proper nouns (nombres de inmuebles, edificios).
- **Forbidden words:** "gratis", "oferta", "descuento", "exclusivo", "urgente" (uppercase), "premium", "elevate", "seamless", "next-gen". No `¡¡`, no `??`.
- **Em-dash (`—`)** allowed in body prose; **banned in button copy and form labels**.

## 10. Accessibility floor (non-negotiable)

- WCAG 2.1 AA across the board; AAA for body text on paper/surface (per §2.3).
- Terracota **never** carries text on light backgrounds where it would fail 4.5:1. Use `color.accent.default` for fills and marks, ink for text on light.
- `warm-amber` **only** on dark backgrounds (hero, CTA, footer). Never on paper — it fails AA there.
- Every interactive element keyboard-reachable in DOM order.
- `focus-visible` ring: `outline: 2px solid #C75D3F; outline-offset: 3px; border-radius: 4px` — **never removed**.
- Decorative images: `alt=""`. Content images: descriptive alt. Property photographs must have alt = "{property.name}, {tipo}, {location}".
- Icon-only controls: `aria-label`.
- Dynamic content (toasts, filter counts, validations): `aria-live="polite"`.
- Form labels always associated (`<label htmlFor>`); placeholder is never the label.
- `prefers-reduced-motion` respected per §6.
- Semantic HTML: `<main id="main-content">`, `<section>` per band, `<article>` per property card, `<nav>`, `<dl>` for property data (dt/dd), `<footer>`.
- Lighthouse Accessibility ≥ 95 before any merge to `main`.

## 11. What this design REJECTS

1. **No blue-corporate** — no Tailwind blue, no navy, no cerulean as accent. The accent is terracota, single, always.
2. **No AI-purple / mesh gradients** — zero `#8b5cf6`, zero aurora / mesh-startup gradients.
3. **No dark-acid** — zero combination of dark background + neon accent. The system's dark surfaces (footer, sidebar, hero band) are warm ink.
4. **No auto dark mode** — single light theme locked. The dark bands (hero, footer, sidebar, CTA) are treatments **within** the light theme, not an OS-driven mode. §12.4.
5. **No glass-everything** — `backdrop-filter: blur` only in: nav on-scroll, hero eyebrow pill, trust-card over photograph, ghost buttons over photograph. **Never** as decoration on regular surfaces.
6. **No shadcn defaults** — zero zinc grays, zero radius-sm on buttons. The whole system is warm parchment, never cold neutral.
7. **No pure-text page** — every view carries at least one real photograph, one structured data block (metric grid, timeline, progress), or a composed pattern. A view of only running text is incomplete work.
8. **No broken image boxes** — every image has a designed fallback (`linear-gradient(135deg, ink-2, ink)`). The design-critic fails the view if it detects a placeholder rendered as visible alt-text or a black box.
9. **No more than one accent** — terracota is the single action/brand color. Gold and olive are status signals only, not decoration.
10. **No raw hex in components** — every value resolves to a semantic token.
11. **No pure-black shadows** — warm-tinted (RGB `26,20,16`).

## 14. Iconografía

Iconos funcionales sólo (nunca decorativos). **Phosphor Icons** (regular weight), 1.5px stroke, `color.accent.default` for interactive icons, `color.fg.default` for structural icons. Sizes: 20×20 in nav / 16×16 in chips and inline / 12×12 in meta.

- `aria-hidden="true"` if purely decorative alongside a text label.
- `aria-label` if icon-only.
- Arrow characters (`→`, `←`) via typography are allowed for back links, pagination cues, CTA buttons — they're type, not icons.

## 15. Real-image & asset strategy (MANDATE)

Domio is a real-estate house. **Real photography is mandatory** — never AI-generated placeholders that render as boxes with cryptic alt text.

- **Hero:** high-quality architectural or lifestyle photograph. Filter: `contrast(.96) saturate(.92) brightness(1.02)` applied globally to all system photography via `.hero-bg img, .property-img img, .cta-wrap img { filter: … }`.
- **Property cards:** exterior photograph of the building. `object-position: center 35%` to favor façade, not sky.
- **Fallback for missing image:** `linear-gradient(135deg, ink-2, ink)` — **never pure black, never a broken alt**.
- **Login / brand surfaces:** photograph as CSS `background-image` in the sidebar with `opacity: 0.18` + warm overlay layer.
- **Avatars:** initials in `linear-gradient(135deg, slate-2, ink-2)` (default) or `linear-gradient(135deg, terracota, gold)` (gestor / owner). Never an empty placeholder.
- **Partner logos:** SVG when available; otherwise partner name in Fraunces italic 16px. Certifications rendered as typography + SVG icon, not as image logos.
- **Verification:** the `design-critic` renders the app and **fails** if any image is showing visible alt text (broken) or a black box.

## 16. Density & composition (MANDATE)

Domio's editorial voice depends on **filled, composed pages** — never a thin corporate template.

- **Vertical section rhythm:** 160px top/bottom for standard sections (`.sec`), 96px for compact (`.sec-tight`), 180px for trust bands. Do not use app spacing (32px) in marketing sections.
- **Asymmetric composition:** at least one per page:
  - Home hero: `1.4fr 1fr`
  - About compare: `1fr 1.05fr`
  - Property grid: `1.3fr 1fr 1fr` (featured `span 2`)
  - FAQ: `1fr 1.2fr`
  - Detail body: `1.6fr 1fr`
  - Contact main: `1.4fr 1fr`
  - CTA band: `1.4fr 1fr`
- **Layout families per page (Home):** full-bleed hero, 4-col how-grid, asymmetric split, 3-col property rail, 4-col metrics + 3-col testi, full-bleed CTA, 2-col FAQ. ≥ 5 distinct skeletons on Home — no two sections repeat.
- **The signature carries:** italic Fraunces numerals appear in ≥ 3 sections of Home (how-steps, metrics, hero trust-card). In detail views, the `numeral-xl` (80px terracota) is the first visual element of the aside.

## 17. States (skeleton / empty / error / loading)

### Skeleton
Dimensions identical to real content (zero layout shift). Shimmer via `bg-position` animated gradient `linear-gradient(90deg, paper-2 25%, paper 50%, paper-2 75%)`, `role="status"` + `aria-hidden="true"`. Reduced-motion: static `bg: paper-2`, no shimmer.

**Per view:**
- Home: hero photo skeleton + trust-card (3 numeral placeholders) + 4 how-step cards + 3 property cards + metrics band.
- Portafolio: filter bar + count line + 12 PropertyCard skeletons.
- Detail: photo hero + infobar (4 cells) + progress numeral + payment steps + 3 related cards.

### Empty
Composed — never bare text. Icon SVG (18×18, terracota) + `heading-sm` (Fraunces) + `body-sm` (fg.subtle) + secondary button (recovery). Example: "Todavía no hay inmuebles en esta zona" + "Ver todo el portafolio".

### Error
Voice per §9 (affirmative, no blame) + visual `border-left: 3px solid accent.default`. No stack traces exposed. No red background. Primary button "Intentar de nuevo" + ghost "Volver al inicio".

### Loading (button)
`aria-busy="true"` on the button. Inline spinner 16×16 (`color: inherit`). Button text remains visible to avoid layout shift.

## 21. Grain / texture

The system uses **SVG fractal-noise grain** on photograph overlays only (hero, CTA) at `opacity: 0.5`, `mix-blend-mode: overlay`, `pointer-events: none`. Never on plain surfaces (cards, panels, canvas) — grain is a photograph companion, not a decorative wallpaper. Under `prefers-reduced-motion`, grain stays (it's static, not animated).

```html
<!-- Reference grain overlay pattern -->
<svg xmlns='http://www.w3.org/2000/svg' width='240' height='240'>
  <filter id='n'>
    <feTurbulence type='fractalNoise' baseFrequency='0.85' numOctaves='2' stitchTiles='stitch'/>
  </filter>
  <rect width='240' height='240' filter='url(#n)' opacity='0.5'/>
</svg>
```

Embedded as `data:image/svg+xml;utf8,…` in the `::after` layer of `.hero-bg` and `.cta-wrap`.
